# Contributing to FinanceFlow

Thank you for your interest in contributing to FinanceFlow!

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/FinanceFlow.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Run tests: `cd backend && mvn test`
6. Commit: `git commit -m "feat: add your feature"`
7. Push: `git push origin feature/your-feature`
8. Open a Pull Request

## Development Setup

See [README.md](README.md#quick-start) for setup instructions.

## Code Style

### Backend (Java)
- Follow standard Java naming conventions
- Use Lombok annotations to reduce boilerplate
- Write unit tests for new features

### Frontend (TypeScript)
- Use functional components with hooks
- Follow the existing component structure
- Use TypeScript types for all props and state

## Commit Messages

Follow [Conventional Commits](https://www.conventionalcommits.org/):

- `feat:` new feature
- `fix:` bug fix
- `docs:` documentation
- `refactor:` code refactoring
- `test:` adding tests
